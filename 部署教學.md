# LINE 行程登記 Bot — Synology NAS 部署教學

## 📁 你需要的檔案

這個資料夾裡有 4 個檔案：
- `app.py` — 主程式
- `requirements.txt` — Python 套件清單
- `Dockerfile` — Docker 容器設定
- `docker-compose.yml` — Docker 啟動設定（⚠️ 需要填入你的金鑰）

另外你還需要準備：
- `credentials.json` — 從 Google Cloud Console 下載的服務帳戶金鑰檔案

---

## 🔧 第一步：填入你的金鑰

用文字編輯器打開 `docker-compose.yml`，把以下值替換成你的真實金鑰：

```
LINE_CHANNEL_SECRET=你的Line_Channel_Secret
LINE_CHANNEL_ACCESS_TOKEN=你的Line_Channel_Access_Token
ANTHROPIC_API_KEY=你的Claude_API_Key
GOOGLE_CALENDAR_ID=你的Google日曆ID
ADMIN_USER_ID=你的Line_User_ID
NAS_EXTERNAL_URL=你的NAS外部網址
```

### 各欄位說明

**LINE_CHANNEL_SECRET / LINE_CHANNEL_ACCESS_TOKEN**
在 LINE Developers Console → 你的 Channel 裡面取得。

**ANTHROPIC_API_KEY**
在 https://console.anthropic.com/ → API Keys 取得。

**GOOGLE_CALENDAR_ID**
1. 開啟 Google 日曆 (https://calendar.google.com/)
2. 左邊找到你的日曆，點旁邊的「⋮」→「設定和共用」
3. 往下找到「整合日曆」區塊
4. 「日曆 ID」就是你要的值（如果是主日曆，通常是你的 Gmail 地址）

**ADMIN_USER_ID**
你的 LINE User ID，用於每天接收行程報表私訊。
取得方式：LINE Developers Console → Basic settings 分頁最下方 → 「Your user ID」
格式像 `U1234567890abcdef...`

**NAS_EXTERNAL_URL**
你 NAS 的外部 HTTPS 網址，用來產生 Excel 報表的下載連結。
例如 `https://yourname.synology.me:58443`（和 Webhook URL 同一個，但不含 `/callback`）

---

## 📂 第二步：把檔案上傳到 NAS

1. 用 File Station 或 SMB 在 NAS 上建立一個資料夾，例如：
   `/docker/line-calendar-bot/`

2. 把以下 5 個檔案放進去：
   - `app.py`
   - `requirements.txt`
   - `Dockerfile`
   - `docker-compose.yml`（已填好金鑰的版本）
   - `credentials.json`（Google 服務帳戶金鑰）

3. 在同一個資料夾內建立 `reports` 子資料夾（用於存放每日報表）：
   - 用 File Station：在 `line-calendar-bot` 資料夾內新增資料夾，命名為 `reports`
   - 或用 SSH：`mkdir -p /volume1/docker/line-calendar-bot/reports`

---

## 🐳 第三步：在 NAS 上啟動 Docker 容器

### 方法 A：用 SSH（推薦）

1. 先在 NAS 的「控制台 → 終端機和 SNMP」啟用 SSH
2. 用終端機（Mac/Linux）或 PuTTY（Windows）SSH 連到 NAS：
   ```
   ssh 你的管理員帳號@你的NAS_IP
   ```
3. 進入專案資料夾：
   ```
   cd /volume1/docker/line-calendar-bot
   ```
4. 建立並啟動容器：
   ```
   sudo docker-compose up -d --build
   ```
5. 確認容器正在運行：
   ```
   sudo docker ps
   ```
   你應該會看到 `line-calendar-bot` 的容器狀態是 `Up`

6. 查看日誌確認沒有錯誤：
   ```
   sudo docker logs line-calendar-bot
   ```

### 方法 B：用 Container Manager（Synology 圖形介面）

如果你不熟 SSH，也可以用 Synology 的 Container Manager：

1. 開啟 Container Manager
2. 到「專案 (Project)」頁面
3. 點「建立」
4. 專案名稱填 `line-calendar-bot`
5. 路徑選 `/docker/line-calendar-bot`
6. 它會自動讀取 docker-compose.yml
7. 點「建立」啟動

---

## 🌐 第四步：設定反向代理（HTTPS）

Line Webhook 要求 HTTPS，所以要設定 Synology 的反向代理：

1. 打開 NAS 的「**控制台**」
2. 進入「**登入入口 → 進階 → 反向代理**」
3. 點「**新增**」
4. 填寫以下資訊：

   **來源（外部）：**
   - 描述：LINE Bot
   - 通訊協定：HTTPS
   - 主機名稱：你的 NAS 的 DDNS 網址（例如 `yourname.synology.me`）
   - 連接埠：58443（或你設定的 HTTPS 對外 port）

   **目的地（內部）：**
   - 通訊協定：HTTP
   - 主機名稱：localhost
   - 連接埠：8000

   ⚠️ 如果你的 58443 port 已被其他服務使用，可以用其他 port（例如 8443），
   但要確保路由器有做 port forwarding。

5. 按「確定」儲存

### 替代方案：用路徑區分

如果你 58443 port 已有其他用途，可以用「自訂標頭」搭配路徑來區分，
或是用不同 port。重點是最終要有一個 HTTPS 的網址能連到 Bot。

---

## 🔗 第五步：設定 Line Webhook

1. 回到 LINE Developers Console (https://developers.line.biz/)
2. 進入你的 Channel → 「Messaging API」分頁
3. 找到「Webhook settings」
4. 在「Webhook URL」填入：
   ```
   https://你的NAS網址/callback
   ```
   例如：`https://yourname.synology.me:58443/callback`

5. 點「Update」
6. 點「Verify」測試連線
   - 如果顯示「Success」就成功了！
   - 如果失敗，請檢查反向代理設定和 port forwarding

7. 確認「Use webhook」是開啟狀態

---

## 🧪 第六步：測試

1. 把你的 Bot 加入 Line 群組
2. 在群組中傳送以「登記」開頭的訊息，例如：

   ```
   登記 3月15日下午兩點到四點，在A棟301會議室，
   總額研商會議，主持人：劉司長，出席人員：理事長(O) 幕僚 張三
   ```

3. Bot 應該會回覆確認訊息，並且 Google 日曆上會出現新事件

---

## 📊 每日報表說明

Bot 每天晚上 6 點會自動私訊管理員（ADMIN_USER_ID）一份 Excel 報表下載連結。

- 報表內容：當日所有新增或修改的行程（不含請假）
- 報表欄位：會議時間、幕僚、會議地點、會議名稱、出席人員
- Excel 檔案存放在 NAS 的 `./reports/` 資料夾
- 自動保留 7 天，過期自動清除
- 如果當天沒有任何行程更新，會收到「今日無新增或修改的行程」的文字通知

> 如果沒有收到報表，請確認：
> 1. `ADMIN_USER_ID` 有正確設定
> 2. `NAS_EXTERNAL_URL` 有正確設定（需要和 Webhook URL 同一個網址，但不含 `/callback`）
> 3. 你有先加 Bot 為好友（Bot 才能私訊你）

## ❓ 常見問題

### Bot 沒有回應？
- 檢查 Docker 容器是否在運行：`sudo docker ps`
- 檢查日誌：`sudo docker logs line-calendar-bot`
- 確認 Webhook URL 設定正確
- 確認反向代理設定正確

### Webhook Verify 失敗？
- 確認 NAS 的 HTTPS 憑證有效
- 確認 port forwarding 設定（路由器需開放對應 port）
- 確認反向代理的目的地 port 是 8000

### 日曆事件沒有建立？
- 確認 Google Calendar ID 正確
- 確認服務帳戶的 email 已加到日曆共用中，且有「變更活動」權限
- 檢查 credentials.json 是否正確放在專案資料夾中

### 每日報表沒有收到？
- 確認 `ADMIN_USER_ID` 設定正確
- 確認你有加 Bot 為好友
- 確認 `NAS_EXTERNAL_URL` 設定正確
- 檢查日誌：`sudo docker logs line-calendar-bot | grep report`

### 想重新啟動 Bot？
```
cd /volume1/docker/line-calendar-bot
sudo docker-compose restart
```

### 想更新程式碼？
```
cd /volume1/docker/line-calendar-bot
sudo docker-compose down
sudo docker-compose up -d --build
```
